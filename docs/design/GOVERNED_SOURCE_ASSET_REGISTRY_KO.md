# 원자료 자산 등록부 설계

## 범위

이 등록부는 기존의 운영 사실과 게시 파일 버전 흐름 옆에 추가되는 합성 데이터 전용 경계다. 원자료는 공식 임대 사실을 직접 덮어쓰지 않는다. 자동 분류 결과는 항상 후보이며, 데이터 담당자의 확인과 선임 담당자의 게시 결정이 각각 필요하다.

## 분류 체계

합성 예시는 `투시도`, `건물 안내자료`, `포트폴리오 안내자료`, `평면도`, `면적 관리표`, `법무 문서` 여섯 범주를 사용한다. 파일명에 있는 날짜는 자료 작성일로만 기록하고 효력 발생일로 추정하지 않는다. 포트폴리오 안내자료는 발행일별 판을 별도 버전 묶음으로 유지하고 `portfolio-wide` 구분 표식을 가진다.

정확히 같은 합성 지문이 다시 들어오면 두 번째 자산을 만들지 않고, 최초 자산에 관찰된 파일명만 추가한다. DWG/DXF 형식은 내용 추출을 지원한다고 가정하지 않으며 직접 확인 대상으로 남긴다.

## 사람의 결정과 제한

1. 결정적 파일 메타데이터 분류가 후보를 만든다.
2. 데이터 담당자가 건물 별칭과 분류를 확인한다. 미해결 별칭이나 다른 건물 선택은 차단된다.
3. 선임 담당자가 게시한다. 평면도는 현재 공식 파일 버전과 연결이 일치해야 한다.

면적 관리표와 법무 문서는 기본 제한 자료다. 모델이나 자동 분류만으로 외부 공유 가능 상태가 될 수 없다. 제한 자료, 지원하지 않는 설계 파일, 중복 자산, 확인 전 후보는 게시할 수 없다.

## 외부 투영 경계

현장 앱과 공개 DTO에는 다음 다섯 조건을 모두 통과한 자산만 포함한다.

- 게시 완료
- 활성 상태
- 게시된 후속 버전이 없는 현재 자료(저장 플래그가 아니라 관계에서 계산)
- 사람의 권한 확인 완료
- 외부 공유 가능

평면도의 경우 현재 공식 파일 버전과의 연결도 다시 확인한다. 새 평면도 후보가 등록되기만 한 상태에서는 기존 게시 평면도가 계속 현재 자료이며, 새 평면도가 게시되면 이전 평면도는 자동으로 이전 자료가 된다.

## 합성 데이터 원칙

저장소에는 `Synthetic_*`, 기존 CFC 데모 자산명과 `synthetic:*` 지문만 둔다. 실제 다운로드 폴더의 파일 바이트, 파일명, 해시, 계약 내용은 코드·시드·문서에 포함하지 않는다. 이 구현은 로컬 `DEMO_MODE=true` JSON 어댑터이며 실제 문서관리시스템 연동을 주장하지 않는다.
