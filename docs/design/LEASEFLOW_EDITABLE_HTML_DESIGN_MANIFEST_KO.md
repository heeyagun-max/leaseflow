# LeaseFlow 편집 가능한 HTML 디자인 결정 기록

## 작업 방식

- 모드: 기존 구현 개선
- 대상: 통합 웹, 모바일 앱, 독립 실행 가능한 HTML 시안
- 핵심 결정: 제목은 현재 위치와 업무 대상을 말하고, 설명은 사용자의 판단 또는 복구에 필요한 순간에만 노출한다.

## 유지·변경·제거

### 유지

- 현재 데이터의 게시 상태, 변경 상태, 승인 상태
- 오류 원인과 다시 시도 방법
- 권한 제한과 가능한 다음 행동
- 건물, 기간, 받는 사람, 자료 버전 등 현재 업무의 실제 메타데이터
- 원자료 등록 → 추출 검토 → 시니어 승인 → 최신정보 게시의 통제 구조
- 임대인 → 여러 건물 → 건물별 주간 보고의 계층
- 사람의 최종 승인 전에는 외부 전달을 완료하지 않는 구조

### 변경

- 페이지 제목 영역은 눈썹말과 제목 또는 제목만 남겼다.
- 안내 자료의 본문처럼 실제 검토해야 하는 내용은 제목 설명에서 검토 항목으로 이동했다.
- 웹과 앱의 핵심 흐름을 각 한 개의 독립 HTML에 재구성했다.
- 역할, 화면, 업무 단계 전환을 HTML 안의 작은 스크립트로 직접 시험할 수 있게 했다.

### 제거

- 제목 바로 아래의 일반적인 사용 설명
- 버튼명을 다시 풀어 쓰는 반복 문장
- 한 번 사용하면 알 수 있는 화면 사용법
- 프로그램 언어, 내부 상태값, 구현 구조를 설명하는 사용자용 문구

## 정보 구조

### 통합 웹

1. 업무 홈
2. 건물정보
3. 자료·업무
4. 주간업무
5. 건물정보 업로드
6. 주간업무 설정
7. 설정·기록

공통 업무 네 개와 관리 업무 세 개를 분리한다. 팀원 역할에서는 관리 업무를 숨기고, 관리 책임자 역할에서는 모두 표시한다.

### 모바일 앱

1. 홈
2. 담당 업무
3. 업무 기록
4. 주간 보고

모바일은 현장 업무의 다음 행동을 우선한다. 건물정보 업로드와 주간업무 설정은 웹 관리 업무로 유지한다.

## 핵심 흐름과 상태

| 흐름 | 주요 상태 | 사용자가 보는 다음 행동 |
|---|---|---|
| 자연어 요청 | 입력 전, 요청 정리, 확인 필요 | 요청 확인 |
| 고객 안내 자료 | 요청 확인, 초안, 승인, 전달 기록 | 만들기, 승인, 확인하고 발송하기 |
| 건물정보 업로드 | 파일 선택, 추출, 변경 검토, 승인 요청 | 반영, 제외, 승인 요청 |
| 최신정보 게시 | 검토 중, 승인됨, 게시됨 | 최신정보로 게시 |
| 주간 보고 | 건물별 준비 전, 내용 확인, 승인, 전달 기록 | 보고 준비, 확인, 승인 |
| 권한 제한 | 접근 불가 | 가능한 메뉴로 돌아가기 |
| 오류 | 불러오기 실패, 저장 실패 | 다시 시도 |

## 시각 체계

- 바탕: 따뜻한 회백색
- 주요 탐색: 짙은 네이비
- 주요 행동: 블루
- 최신·완료: 그린
- 확인 필요: 앰버
- 차단·오류: 레드
- 모서리: 8–18px 범위의 제한된 라운드
- 그림자: 계층 구분에 필요한 낮은 강도만 사용
- 제목 크기: 웹 34px, 앱 27px
- 최소 조작 영역: 44px
- 애니메이션 감소 설정을 존중한다.

## 업무 홈 Bento 리팩터링

- 홈의 제목, 자연어 요청, 업무 현황을 서로 떨어진 세로 영역이 아니라 하나의 12컬럼 Bento 그리드로 통합했다.
- 데스크톱은 업무 중요도에 따라 4·8·5·3·4·7컬럼으로 비대칭 배치한다.
- 모바일은 제목과 요청창을 전체 너비로, 네 업무 카드를 2열로 배치해 397×844와 375×667 화면에서 스크롤 없이 첫 화면을 완결한다.
- 자연어 요청창의 설명 문장은 시각적으로 제거하고 접근성 이름만 유지했다. 마이크와 위쪽 화살표만 노출하며, Enter로 바로 요청할 수 있다.
- 홈의 네 업무 단위는 `진행 중인 업무`, `처리한 요청`, `건물 정보 최신 업데이트`, `주간업무 보고 생성`으로 고정했다.
- 홈 전용 색상은 `#FBFBFD`, `#1D1D1F`, `#0071E3`, `#E9E9EB`, `#979799`, `#CACACC`를 사용한다.
- 카드 진입은 12px 아래에서 순차적으로 나타나고, 포인터 호버는 3px 상승과 약한 확대만 적용한다. 모든 전환은 250ms 이내이며 애니메이션 감소 설정에서는 제거된다.
- 조작 버튼의 최소 영역은 모바일에서도 44px을 유지하고, 한글 제목은 글자 중간에서 끊기지 않게 처리했다.
- 모션과 비대칭 카드 배치는 Design Isms의 Bento Grid 사례를 참고하되, 업무 우선순위와 접근성 기준에 맞춰 절제했다.

## 편집 파일

- `docs/design/editable/LeaseFlow_Web_Editable.html`
- `docs/design/editable/LeaseFlow_App_Editable.html`

두 파일은 외부 라이브러리와 이미지 없이 실행된다. 색상과 간격은 문서 상단의 CSS 변수, 화면 내용은 `body`, 동작은 문서 하단의 스크립트에서 바로 수정할 수 있다.

## 검증 기준

- 제목 바로 아래에 일반 설명 문장이 없어야 한다.
- 실제 판단에 필요한 데이터와 복구 메시지는 남아 있어야 한다.
- 웹의 일곱 메뉴와 앱의 네 메뉴를 모두 이동할 수 있어야 한다.
- 자연어 요청, 파일 선택, 업무 단계 전환, 주간 보고 확인 동작이 작동해야 한다.
- 작은 화면에서 가로 스크롤 없이 사용할 수 있어야 한다.
- 실제 제품의 타입 검사, 테스트, 관리자 웹 빌드가 통과해야 한다.

## 검증 결과

- 웹 HTML: 일곱 메뉴 이동, 자연어 검색 결과, 역할에 따른 관리 메뉴 숨김, 파일 선택 동작 확인
- 앱 HTML: 네 메뉴 이동, 요청 시작, 승인 단계 진행, 전달 완료, 주간 보고 이동 확인
- 반응형: 웹 1440px, 모바일 397×844 및 375×667에서 가로 넘침 없음
- 홈 첫 화면: 두 모바일 크기 모두 문서 높이와 화면 높이가 일치하고 네 업무 카드가 전부 노출됨
- 자연어 요청: Enter 제출, 음성 입력 버튼, 요청 결과 노출 확인
- 실제 관리자 웹: 홈 카드에서 주간업무 화면으로 이동 확인
- 실제 제품: 전체 워크스페이스 타입 검사 통과
- 자동 테스트: 212개 통과
- 관리자 웹: 32개 정적 페이지 생성 포함 프로덕션 빌드 통과
- 패키지 검사: 통과
