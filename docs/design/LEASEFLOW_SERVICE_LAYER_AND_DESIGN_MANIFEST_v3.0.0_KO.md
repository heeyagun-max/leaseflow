## 2. 디자인 명제

LeaseFlow는 부동산 대시보드가 아니라 **출처가 있는 정보가 사람의 판단을 거쳐 외부 업무가 되는 정밀한 업무 대장**이다. 시각적 콘셉트는 `Quiet Operations Ledger / 정밀한 업무 대장`으로 고정한다.

- 따뜻한 종이색 바탕은 오래 읽는 업무의 피로를 줄인다.
- 먹색 사이드바는 관리자 영역의 폐쇄성과 안정감을 만든다.
- 청색은 선택과 다음 행동에만 쓴다.
- 녹색은 완료, 황갈색은 확인 필요에만 쓴다.
- 화면의 기본 읽기 방향은 `요청 또는 출처 → 현재 정보 → 차이 → 사람의 결정 → 외부 결과`다.
- 카드는 정보의 경계를 설명할 때만 사용하며, 박스 안의 박스는 만들지 않는다.

## 4. 화면 계층

### 관리자 웹

고정 탐색 순서는 `대시보드 → 원자료 → 검토·게시 → 건물 → 보고그룹 → 보고일정 → 사용자·권한 → 발송·감사`다.

핵심 검토 화면은 세 구역으로 구성한다.

1. 상단 요약: 검토 대기, 오늘 게시, 보고 준비처럼 지금의 운영량만 짧게 보여준다.
2. 증거 작업대: 원자료와 변경 전·후를 넓은 영역에서 읽는다.
3. 결정 레일: 담당자, 진행 단계, 확인 필요 항목과 한 개의 주 행동을 둔다.

### 모바일 앱

고정 하단 탭은 `홈 → 담당 업무 → 업무 기록 → 주간 보고`다. 건물은 탭이 아니라 검색·필터·업무 결과의 속성이다.

- 홈의 첫 기능은 자연어·음성 요청이다.
- 담당 업무는 현재 필요한 행동을 한 번에 하나만 강조한다.
- 발송 검토는 요청 이해, 최신 승인 정보, 제외·누락, 파일, 메일, 수신자를 한 흐름에서 읽는다.
- 외부 발송 확인은 가장 마지막에 한 번만 한다.
- 기록에서 뒤로 가면 검색어·필터·정렬·스크롤·선택 탭을 복원한다.

## 5. 상태 표현 계약

| 도메인 의미 | 사용자 표현 | 색상 | 허용 행동 |
| --- | --- | --- | --- |
| 아직 시작 전 | 준비 전 | 중립 | 첫 단계 시작 |
| AI 또는 규칙이 만든 후보 | 확인 필요 | 청색 또는 황갈색 | 확인·수정·보완 요청 |
| 주니어 확인 완료 | 1차 확인 완료 | 녹색 | 선임 검토 |
| 선임 승인 완료 | 최종 승인 완료 | 녹색 | 게시 또는 발송 준비 |
| 외부 사용 가능 | 현장 공유 중 / 최신 정보 | 녹색 | 조회·패키지 구성 |
| 이전 버전 | 이전 정보 | 중립 | 이력 조회만 |
| 기준 변경으로 무효 | 새 초안 필요 | 황갈색 | 최신 기준으로 다시 준비 |
| 실패 | 완료하지 못함 | 적색 | 안전한 재시도 |

내부 열거형, 코드, API, JSON, 내부 ID, `published`, `superseded`, `pending_review`, `model`, `debug`, `localhost`는 기본 화면에 노출하지 않는다.

## 6. 시각 시스템 결정

- 테마: Quiet Premium Neutral
- 배경: 저채도 종이 질감. 실제 구현은 성능과 가독성을 위해 CSS의 미세한 점·선 패턴만 사용한다.
- 타이포그래피: 한국어 가독성이 높은 refined grotesk. 제목은 굵기와 크기로만 대비한다.
- 구조: 스위스식 그리드와 비대칭 작업대의 결합.
- 대표 구성요소 4개: 세로 리듬 선, 원자료 고정 프레임, 전후 비교 행, 결정 타임라인.
- 움직임 2개: 단계 전환의 짧은 fade-through, 목록의 절제된 staggered reveal. `prefers-reduced-motion`에서는 제거한다.
- 모서리: 6 / 9 / 12 / 16px. 상태 외에는 pill을 사용하지 않는다.
- 터치 목표: 최소 44×44px.

## 7. 생성 시안 분석과 구현 번역

### 관리자 웹

- 보존: 먹색 고정 사이드바, 60/40 증거·결정 작업대, 상단 얇은 운영 요약, 출처 기록 레일.
- 번역: 이미지 속 평면도는 실제 데이터가 없는 상태에서 꾸며내지 않고, 합성 원자료 프리뷰 또는 명시적 자리표시자로 제공한다.
- 제외: 과도한 아이콘, 가짜 알림, 실제로 존재하지 않는 기록 수치.

### 모바일 홈

- 보존: 첫 화면 자연어 요청, 세로 청색 리듬 선, 다음 업무와 최근 결과, 고정 4탭.
- 번역: 예시 문장은 현재 합성 요청 데이터와 일치시킨다.
- 제외: 장식적 기기 프레임과 실제 운영에 필요 없는 아바타 효과.

### 모바일 발송 검토

- 보존: 한 열의 검토 흐름, 최신 승인 정보와 제외 항목, 파일·메일·수신자, 마지막 한 번의 승인.
- 번역: 현재 상태기계의 draft/edit/approved/sent 단계에 따라 필요한 구역과 행동만 노출한다.
- 제외: 승인되지 않은 정보나 존재하지 않는 파일 이름의 하드코딩.

## 8. 반응형·접근성 기준

- 관리자 검증 크기: 1440×1024, 1280, 768, 375px.
- 모바일 검증 크기: 390×844, 430×932, 375px, 200% 확대.
- 모든 주요 화면은 키보드 탐색, 명확한 포커스, 색상 외 상태 텍스트, 오류 후 복구 행동을 제공한다.
- 사이드바는 작은 화면에서 가로 스크롤 메뉴가 아니라 축약된 상단 탐색으로 전환한다.
- 고정 하단 탭은 콘텐츠를 가리지 않도록 안전 영역과 하단 패딩을 함께 확보한다.

## 9. 금지 결정

- 마케팅 히어로, 가짜 KPI, 지도 중심 대시보드, 부동산용 청록색 템플릿을 사용하지 않는다.
- 유리 효과, 네온, 무지개 또는 보라-파랑 AI 그라디언트를 사용하지 않는다.
- 사람 승인과 코드 게이트를 AI 문안으로 대체하지 않는다.
- 생성 시안의 존재하지 않는 데이터를 제품 사실처럼 구현하지 않는다.
- 기존 도메인 상태와 테스트를 깨뜨리는 순수 시각용 재구축을 하지 않는다.
