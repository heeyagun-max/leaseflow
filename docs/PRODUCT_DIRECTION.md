# Revised Product Direction

## Product purpose

LeaseFlow should become a small-team operating system for commercial leasing work, not merely a hackathon email generator.

## Product surfaces

### A. Data Admin Web
Restricted to designated users.

Responsibilities:

- upload source material;
- extract structured change candidates with GPT-5.6;
- compare against the current published version;
- junior review and correction;
- senior approval or rejection;
- publish official operational records and external-use files;
- preserve source, page, uploader, reviewer, effective date, and version chain.

### B. Mobile Operations App
Used by the LM manager and team.

Responsibilities:

- receive/import call, email, message, or natural-language requests;
- identify building, floor/space, requested facts/files, recipient, and deadline;
- query only current published records;
- show recent changes and unresolved exceptions;
- prepare approved attachments and an external email draft;
- accept natural-language revisions as a scoped patch;
- require approval before send;
- write activity memory for weekly reporting.

### C. Shared backend

Responsibilities:

- organization/team/user roles;
- building and portfolio access;
- source and publication workflow;
- versioned space, terms, and files;
- activity and task memory;
- weekly report state;
- recipient configuration;
- audit and output logs.

## Long-term rollout

1. synthetic hackathon MVP;
2. husband + small team pilot with anonymized/manual imports;
3. approved Outlook integration and stronger identity controls;
4. expanded buildings, templates, and vendor-source parsers;
5. company-approved production deployment.
