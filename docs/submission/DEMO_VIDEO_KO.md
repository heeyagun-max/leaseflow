## 내레이션 원고

LeaseFlow는 임대인에게서 계속 들어오는 자료를, 현장 팀이 안전하게 사용할 수 있는 최신 운영 정보로 바꾸는 폐쇄형 업무 시스템입니다.

먼저 관리자 웹의 원자료 등록부입니다. 건물 안내서, 면적표, 평면도 같은 파일이 들어오면 파일 종류와 건물 후보를 분류하고, 동일 자료와 새 버전을 구분해 자산으로 기록합니다. 화면의 파일명과 내용은 모두 합성 데모 데이터입니다.

AI가 찾은 변경은 곧바로 공식 정보가 되지 않습니다. 데이터 담당자가 출처와 네 가지 변경안을 확인한 뒤, 선임 검토자가 게시해야 합니다. 게시가 끝나면 5층 임대 가능 면적은 삼백 평에서 이백 평으로, 평면도는 버전 일이 아닌 버전 이가 현재본이 됩니다. 이전 버전은 이력으로 남지만 외부 자료에서는 차단됩니다.

모바일 운영 화면은 게시된 현재 정보만 읽습니다. 합성 통화 요청에서 건물, 층, 면적, 조건, 주차, 최신 평면도와 기한을 추출하고, 이백 평과 현재 평면도 버전 이로 안내 자료를 준비합니다. 수신자는 설정된 그룹에서만 가져오며 담당자가 내용을 승인하기 전에는 발송할 수 없습니다. 여기서 발송은 실제 이메일이 아닌 샌드박스 기록입니다.

같은 활동은 건물별 주간 보고서로 이어집니다. ‘협의 중인 면적 변동 있는지 확인해’라는 요청에 대해, 시스템은 앱 활동과 합성 메일 근거를 연결한 변경안만 제안합니다. 사용자가 근거를 확인하고 반영한 뒤 다시 승인해야 보고서가 완성됩니다.

이 데모의 핵심은 생성형 AI가 결정을 대신하는 것이 아닙니다. AI는 분류, 추출, 초안과 패치를 만들고, 결정적 코드가 권한, 버전, 현재성, 수신자와 발송 조건을 강제합니다. 모든 데이터는 합성이며 실제 아웃룩, 회사 로그인, 전화망, 외부 발송과 연결되지 않습니다. LeaseFlow는 빠른 자동화와 사람의 책임을 하나의 감사 가능한 흐름으로 묶습니다.
