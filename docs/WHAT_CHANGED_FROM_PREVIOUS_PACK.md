# What Changed from the Previous Pack

## Previous assumption

- The official source-approval database was outside LeaseFlow.
- LeaseFlow was a single responsive web/PWA system of action.
- The hackathon workflow started from already-published demo data.

## Revised direction

- The restricted Data Admin Web is now part of LeaseFlow.
- The Mobile Operations App is a separate product surface for LM managers and team members.
- Both surfaces share one governed backend, version model, and audit trail.
- The hackathon MVP begins with source extraction, junior confirmation, and senior publication before the mobile request uses the data.
- Mobile is scaffolded with Expo / React Native and a web target for judge access.
- Production Outlook, carrier-call APIs, SSO, and real company data remain post-hackathon pilot work.

## Unchanged rules

- Communications never overwrite official records.
- Terms, available area, floors, parking, and drawings are versioned.
- Only current published externally shareable records enter external output.
- Human approval is required before external send.
- Weekly reports are building-specific, external landlord reports.
