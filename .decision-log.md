# LeaseFlow UX Decision Log

## 2026-07-19 — G008 관리자 경험 기준본

### Context

G008 구현 전 관리자 웹은 검증·게시·운영 정보·감사를 `/`의 장문 페이지에 결합했고, 전역 메뉴 다수를 같은 페이지 앵커로 연결했다. G008은 기존 도메인/권한/게시 불변식을 유지하면서 실제 사용자 흐름, 복구, 반응형, 접근성을 기준으로 관리자 IA와 시각 계약을 재정의했고 현재 구현은 이 결정을 반영한다.

### Decisions

1. **두 기준본 분리:** `DESIGN.md`는 시각 정체성과 token, `EXPERIENCE.md`는 IA·행동·상태·접근성·여정을 소유한다. 목업과 구현 세부가 충돌하면 기준본이 우선하되 도메인 규칙은 코드와 `docs/DATA_GOVERNANCE.md`가 우선한다.
2. **시각 방향:** `Quiet Evidence Ledger`. warm paper + ink rail + single cobalt + semantic state. Pretendard Variable 단일 refined grotesk, H1 24–28px, border/surface hierarchy.
3. **디자인 다이얼:** variance 3/10, motion 1/10, product density D6, visual density 7/10. 반복적이고 고위험인 운영 업무의 위치 기억과 검토 밀도를 우선한다.
4. **Canonical IA:** 오늘의 업무 `/`, 원자료 `/sources`, 변경 검토 `/changes`, 승인·게시 `/publishing`, 운영 정보 `/operations`, 임대인 보고 `/reports`, 설정·기록 `/settings`.
5. **Real-route gate:** route module과 상태 표면이 구현되지 않으면 메뉴도 노출하지 않는다. 전역 nav에서 페이지 앵커를 독립 목적지로 사용하지 않는다.
6. **페이지 문법:** task queue → evidence/comparison → decision rail. 원자료는 dense table/list + detail. page당 h1 하나, 업무별 h2, 항목별 h3.
7. **한 행동:** 현재 단계의 primary action 하나만 제공한다. 미래 단계는 disabled button이 아니라 역할·조건을 설명하는 handoff state다.
8. **데모 분리:** 역할 전환과 초기화는 업무 결정 레일이 아닌 별도 DemoTools landmark다.
9. **언어 경계:** raw enum/JSON/API 오류/내부 ID/영문 fallback/근거 없는 confidence를 노출하지 않는다. 타입별 exhaustive 한국어 label mapper와 `Intl` `ko-KR`를 사용한다.
10. **상태·복구:** loading/empty/error/retry/success/permission/read-only/stale/conflict/disabled가 원인·영향·복구 행동을 가진다.
11. **반응형:** `≥1024` sidebar, `768–1023` compact topbar+drawer, `<768` appbar+drawer. global nav horizontal scroll과 200% 본문 2D scroll을 금지한다.
12. **접근성:** 44px targets, unique IDs, skip target, scroll-margin, route h1 focus, light/rail focus ≥3:1, reduced motion, forced colors, semantic elements.
13. **모션:** CSS only. press 140ms, state 180–220ms, panel ≤360ms. JS motion dependency 없음.
14. **장식 제거:** gradients, glass, fake floorplan, nested cards, marketing hero, KPI theater, Georgia/display serif를 제거한다.
15. **합성 자료 등록 경계:** `/sources/new`는 실제 업로드 폼이 아니라 저장소에 포함된 합성 예시 자료의 등록 범위와 다음 단계를 설명하는 안내 route다. 외부 문서 저장소나 회사 시스템 연동을 주장하지 않는다.
16. **역할별 행동:** 데이터 담당자는 원자료 분류와 변경 후보 1차 확인, 선임 검토자는 승인·게시, 임대 관리 책임자는 게시된 운영 정보와 보고 업무를 담당한다. 허용되지 않은 단계는 mutation control을 제거하고 읽기·권한 안내를 표시한다.

## 2026-07-19 — G009 감사 기록 fail-closed 결정

`/settings/audit`는 여러 endpoint의 결과를 하나의 감사 표면으로 합성하므로, 보고 endpoint를 포함한 필수 조회 하나라도 실패하면 부분 기록을 보여주지 않는다. 화면은 fail closed로 모든 감사 항목을 숨기고 안전한 오류 설명과 같은 영역의 `다시 시도`만 제공한다. 재시도가 성공한 뒤에만 전체 감사 기록을 복원한다.

현재 최종 검증은 Admin 78/78, 전체 147/147 tests, 5/5 workspace typecheck, Admin production build 24 pages, final 14-route viewport smoke를 통과했으며 console error, HTTP failure, horizontal overflow는 모두 0이다. 최종 code review는 APPROVE, architect review는 CLEAR다. 권위 판정은 `artifacts/visual-qa/g008/G008_FINAL_QA.md`에 기록한다.
